# 07 - 编码规范

> **权威参考**: `.github/copilot-instructions.md` 中的 Code Quality & Style 和 Anti-Redundancy Rules 章节是编码规范的权威定义。

## 反冗余规则（强制）

| 规则 | 说明 |
|------|------|
| 复用现有工具类 | 如果 `AtomUI.Core`、`AtomUI.Controls.Shared` 或 `AtomUI.Controls` 中已存在 utility/converter/helper，**必须导入使用**，不得创建重复 |
| 复用共享接口 | 如果 `ISizeTypeAware`、`IInputControlStatusAware`、`IWaveSpiritAwareControl` 等接口已存在，**必须实现**，不得定义新接口 |
| 复用共享属性 | 使用 `AddOwner<T>()` 模式复用 `StyledProperty`（如 `SizeTypeControlProperty.SizeTypeProperty.AddOwner<Button>()`） |
| 复用伪类常量 | 创建伪类前**必须先检查** `StdPseudoClass` 常量 |
| Token 值不可硬编码 | Token 值必须从 `SharedToken` 派生，**禁止**硬编码颜色、尺寸、间距 |
| 搜索先行 | 生成新代码前，**必须搜索**类似实现并镜像其结构、命名和约定 |

## 命名约定

### 通用规则

| 类型 | 规范 | 示例 |
|------|------|------|
| 类 | PascalCase | `ButtonToken`, `ThemeManager` |
| 接口 | PascalCase + I 前缀 | `IThemeManagerBuilder`, `ILanguageProvider` |
| 枚举 | PascalCase | `ButtonType`, `SizeType`, `Shape` |
| 枚举值 | PascalCase | `Default`, `Primary`, `Dashed` |
| 属性 | PascalCase | `ButtonType`, `IsDanger`, `IsLoading` |
| 方法 | PascalCase | `Calculate()`, `OnApplyTemplate()` |
| 事件 | PascalCase + On 前缀 | `OnClick`, `OnValueChanged` |

## 代码组织

### 文件结构（每个控件）

```
{ControlName}/
├── {ControlName}.cs              # 1. DependencyProperty 定义
│                                  # 2. CLR 属性
│                                  # 3. 静态构造函数
│                                  # 4. 实例构造函数
│                                  # 5. OnApplyTemplate
│                                  # 6. 事件处理
│                                  # 7. 属性变更回调
│                                  # 8. 辅助方法
├── {ControlName}Token.cs         # Token 定义
└── Themes/
    ├── {ControlName}Theme.axaml  # XAML 主题
    ├── {ControlName}Theme.cs     # Code-behind
    └── {ControlName}ThemeConstants.cs  # PART 常量
```

### Partial Class 使用

控件类跨项目使用 `partial class`：

- `AtomUI.Controls`：属性定义、事件定义
- `AtomUI.Desktop.Controls`：实现逻辑、模板交互

### XAML 组织

主题 XAML 文件内部组织顺序：

1. `ControlTheme` 根元素（含 `x:Class` 和 `TargetType`）
2. `ControlTemplate` 定义（按主要变体分组）
3. 公共 `Setter`（默认值）
4. `Style Selector`（按属性/状态分组）
5. 模板内元素样式（`/template/` 选择器）

## 注释与文档规范

### XML 文档注释

公共 API 使用 XML 文档注释：

```csharp
/// <summary>
/// 按钮类型
/// </summary>
public enum ButtonType { ... }

/// <summary>
/// 获取或设置按钮类型
/// </summary>
public ButtonType ButtonType { get; set; }
```

### 生成文件注释

Source Generator 生成的文件头部包含：

```csharp
// <auto-generated/>
// 此代码由 AtomUI.Generator 自动生成，不要手动编辑
```

### 内部类标记

内部使用的类标记为 `internal`：

```csharp
internal class ButtonThemeConstants { ... }
internal class DesktopControlThemesProvider : ControlThemesProvider { ... }
```

## 设计模式使用

| 模式 | 使用场景 | 示例 |
|------|----------|------|
| Builder | 主题构建 | `ThemeManagerBuilder` |

## 异常处理

### 模式

- 控件代码中通常不抛出异常，而是使用默认值或静默处理
- `OnApplyTemplate` 中使用 `?.` 空条件运算符
- Token 计算中不进行异常处理（依赖 GlobalToken 的正确初始化）

### 资源管理

- 控件中的事件订阅在 `OnApplyTemplate` 中建立
- 使用 Avalonia 的响应式订阅（`Changed.Subscribe`）而非手动 += / -=
- 模板元素通过 `e.NameScope.Find<T>()` 获取

## Git 提交规范

> 详细规范见 `.agents/skills/commit-msg/references/commit-message-convention.md`

### 格式

```
<type>(<scope>): <subject>

[optional body]

[optional footer(s)]
```

### Type 清单

| Type | 用途 |
|------|------|
| `feat` | 新控件、属性、API 或用户可见行为 |
| `fix` | Bug 修复 |
| `refactor` | 代码重构（无行为变更） |
| `docs` | 仅文档变更 |
| `style` | 代码格式化（无逻辑变更） |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `build` | 构建系统或依赖变更 |
| `chore` | 维护性工作 |
| `ci` | CI/CD 配置 |
| `gallery` | Gallery/ShowCase 变更 |
| `release` | 版本发布 |

### Scope 清单

| 类别 | 示例 |
|------|------|
| 控件名 | `Button`, `Select`, `Tag`, `DatePicker`, `TreeView` |
| 基础设施 | `TokenSystem`, `Theme`, `Motion`, `WaveSpirit`, `Core`, `Icons`, `Generator` |
| 横切关注点 | `deps`, `scripts`, `Packaging`, `MemoryLeak` |

### 关键规则

- `type` 和 `scope` **必须英文**
- `subject` 优先英文，中文也可接受
- 祈使语气："add" 而非 "added"
- 首字母小写，不以句号结尾
- Header 行 ≤ 72 字符
- 跨层变更优先**分层提交**

### 示例

```
feat(Button): add danger style variant for ghost button
fix(Select): 修复多选模式下点击后下拉框意外关闭
refactor(Avatar): extract AbstractAvatar to AtomUI.Controls
docs(Core): add three-tier architecture documentation
chore(deps): bump Avalonia from 11.3.1 to 11.3.2
```

## Avalonia 特定规范

### DependencyProperty 注册

```csharp
// StyledProperty（可被样式设置，最常用）
public static readonly StyledProperty<ButtonType> ButtonTypeProperty =
    AvaloniaProperty.Register<Button, ButtonType>(nameof(ButtonType), ButtonType.Default);

// DirectProperty（高性能，直接绑定 backing field）
public static readonly DirectProperty<MyControl, string> ValueProperty =
    AvaloniaProperty.RegisterDirect<MyControl, string>(nameof(Value), o => o.Value);
```

### 属性变更订阅

```csharp
// 静态构造函数中订阅
static Button()
{
    IsDangerProperty.Changed.Subscribe(OnIsDangerChanged);
}

// 回调方法
private static void OnIsDangerChanged(AvaloniaPropertyChangedEventArgs e)
{
    var button = (Button)e.Sender;
    button.PseudoClasses.Set(DangerPseudoClass, button.IsDanger);
}
```

### PseudoClass 定义

```csharp
// 自定义伪类
private static readonly PseudoClass DangerPseudoClass = new(":danger");
private static readonly PseudoClass LoadingPseudoClass = new(":loading");

// 设置伪类状态
PseudoClasses.Set(DangerPseudoClass, isDanger);
```

### XAML 绑定模式

```xml
<!-- 模板绑定（最常用） -->
<Border Background="{TemplateBinding Background}" />

<!-- 相对源绑定（访问 TemplatedParent 属性） -->
<Binding RelativeSource="{RelativeSource TemplatedParent}" Path="Icon" />

<!-- Token 资源绑定 -->
<Setter Property="Foreground" Value="{atom:ButtonTokenResource DefaultColor}" />

<!-- 多值绑定 -->
<MultiBinding Converter="{x:Static atom:ButtonTheme.IconVisibleConverter}">
    <Binding RelativeSource="{RelativeSource TemplatedParent}" Path="Icon" />
    <Binding RelativeSource="{RelativeSource TemplatedParent}" Path="IsIconVisible" />
</MultiBinding>
